# Лабораторная работа №1: Оркестрация многоконтейнерного веб-стека в Podman (v2.0.0.0)

Проект представляет собой отказоустойчивую, масштабируемую и безопасную изолированную инфраструктуру веб-сервера с балансировщиком нагрузки, развернутую с использованием Podman Compose в rootless-режиме на ОС Fedora.

В версии **v2.0.0.0** реализована полноценная защита трафика (SSL-терминация), автоматическое форсирование HTTPS-соединения и динамическое взаимодействие веб-узлов с СУБД.

## 🏗 Архитектура проекта
Инфраструктура состоит из пяти независимых контейнеров, разделенных на две изолированные сети:

1. **proxy (HAProxy 3.4-alpine)** — входная точка и SSL-терминатор. Принимает внешний незащищенный трафик на порту `8080` и автоматически редиректит его на безопасный порт `8443` с использованием самоподписанного SSL-сертификата. Распределяет запросы между веб-нодами по алгоритму Round Robin. На порту `8404` доступна панель статистики. Подключен к `frontend-net`.
2. **web1 и web2 (Nginx 1.26-alpine)** — два параллельных независимых веб-сервера. Работают как прокси-форвардеры: принимают трафик от HAProxy и перенаправляют динамические PHP-запросы на бэкенд. Подключены к сетям `frontend-net` и `backend-net`, выступая мостом.
3. **php-fpm (PHP 8.3-fpm-alpine)** — выделенный интерпретатор PHP. Изолирован в сети `backend-net`. Обрабатывает динамический код, выполняет запросы к базе данных через встроенный драйвер PDO MySQL и возвращает сгенерированную страницу с уникальным ID обработавшей ноды Nginx.
4. **db (MariaDB 10.11)** — СУБД. Полностью изолирована во внутренней сети `backend-net`. Данные сохраняются в постоянном волюме `db_data`.

## 📁 Структура каталогов
* `haproxy/` — конфигурация балансировщика (`haproxy.cfg`).
* `haproxy/certs/` — файлы ключей и самоподписанного SSL-сертификата (`server.pem`).
* `nginx/` — настройки виртуальных хостов Nginx (`default.conf`) в режиме проксирования PHP.
* `php-fpm/` — `Dockerfile` для сборки кастомного образа PHP с расширениями `pdo_mysql`.
* `web-site/` — динамический исходный код веб-страниц (`index.php`).
* `podman-compose.yml` — главный декларативный манифест оркестрации.

## 🚀 Быстрый старт

### 1. Подготовка SSL-сертификата на хосте (Fedora)
Сгенерируйте и объедините ключи для HAProxy:
```bash
mkdir -p haproxy/certs
openssl req -x509 -newkey rsa:2048 -nodes -keyout haproxy/certs/key.pem -out haproxy/certs/cert.pem -days 365 -subj "/CN=localhost"
cat haproxy/certs/cert.pem haproxy/certs/key.pem > haproxy/certs/server.pem
rm haproxy/certs/key.pem haproxy/certs/cert.pem
```

### 2. Запуск инфраструктуры со сборкой образов
```bash
podman-compose up -d --build
```

### 3. Проверка статуса запущенных контейнеров
```bash
podman-compose ps
```

## 📊 Проверка работоспособности
* **Просмотр сайта (HTTPS)**: Откройте в браузере `https://localhost:8443`. Из-за самоподписанного сертификата подтвердите исключение безопасности. При обновлении страницы (F5) имя сервера будет циклически меняться между `web1` и `web2`.
* **Автоматический редирект**: Введите в браузере или через curl `http://localhost:8080`. Система автоматически выполнит 301-редирект на порт `https://localhost:8443`.
* **Панель мониторинга HAProxy**: Доступна в браузере по адресу `http://localhost:8404`.

## 🔒 Особенности реализации (Rootless & Security)
* Проект полностью работает в rootless режиме без прав суперпользователя.
* Для монтирования конфигурационных файлов использован флаг `:Z` и режим `:ro` (Read-Only) для защиты от изменений изнутри контейнеров.
* **Внимание**: Для папки проекта `./web-site/` применен разделяемый флаг **`:z`** (строчная буква), что обеспечивает одновременный безопасный доступ со стороны SELinux для нескольких контейнеров (`web1`, `web2`, `php-fpm`).
