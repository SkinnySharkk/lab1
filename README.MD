# Лабораторная работа №1: Оркестрация многоконтейнерного веб-стека в Podman

Проект представляет собой отказоустойчивую изолированную инфраструктуру веб-сервера с балансировщиком нагрузки, развернутую с использованием **Podman Compose** в безопасном rootless-режиме на операционной системе Fedora.

## 🏗 Архитектура проекта

Инфраструктура состоит из трех независимых сервисов, разделенных на две изолированные сети:
1. **proxy (HAProxy 3.4-alpine)** — входная точка. Принимает внешний трафик на порту `8080` и перенаправляет на веб-сервер. На порту `8404` доступна панель статистики. Подключен к `frontend-net`.
2. **web (Nginx 1.26-alpine)** — веб-сервер. Отдает статическую страницу сайта. Подключен к сетям `frontend-net` и `backend-net`, выступая мостом.
3. **db (MariaDB 10.11)** — СУБД. Полностью изолирована во внутренней сети `backend-net`. Данные сохраняются в постоянном волюме `db_data`.

## 📁 Структура каталогов
* `haproxy/` — конфигурационные файлы балансировщика (`haproxy.cfg`).
* `nginx/` — настройки виртуальных хостов веб-сервера (`default.conf`).
* `web-site/` — исходный код веб-страниц (`index.html`).
* `podman-compose.yml` — главный декларативный манифест оркестрации.

## 🚀 Быстрый старт

1. Склонируйте репозиторий:
```bash
git clone <ссылка_на_ваш_репозиторий>
cd lab1
```

2. Запустите инфраструктуру в фоновом режиме:
```bash
podman-compose up -d
```

3. Проверьте статус запущенных контейнеров:
```bash
podman-compose ps
```

## 📊 Проверка работоспособности
* **Просмотр сайта:** выполните команду `curl http://localhost:8080` или откройте этот адрес в браузере.
* **Панель мониторинга HAProxy:** доступна в браузере по адресу `http://localhost:8404`.

## 🔒 Особенности реализации (Rootless & Security)
* Проект полностью работает в **rootless** режиме без прав суперпользователя.
* Для обхода ограничений привилегированных портов Linux внутренний порт HAProxy изменен на `8080`.
* При монтировании локальных папок хоста использован флаг `:Z` для автоматической настройки контекстов безопасности **SELinux** на Fedora.
* Все конфигурационные файлы примонтированы в режиме `:ro` (Read-Only) для защиты от изменения изнутри контейнеров.
