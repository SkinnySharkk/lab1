# Лабораторная работа №1: Оркестрация многоконтейнерного веб-стека в Podman (v3.0.0)

Проект представляет собой отказоустойчивую, масштабируемую и безопасную изолированную инфраструктуру веб-сервера с балансировщиком нагрузки, развернутую с использованием Podman Compose в rootless-режиме на ОС Fedora.

В версии **v3.0.0** проект полностью доведен до эталонного состояния, соответствующего **12-факторной методологии (12-Factor App)** и лучшим production-практикам контейнеризации.

**Ключевые изменения v3.0.0:**
* **Stateless-архитектура (IV фактор)**: Сессии пользователей вынесены из локальных файлов PHP в оперативную память **Redis**. Приложение готово к бесконечному горизонтальному масштабированию.
* **Изоляция конфигурации (III фактор)**: Все секреты и пароли полностью вынесены в файл `.env` и исключены из Git-репозитория.
* **Наблюдаемость (Health-checks)**: Для всех 7 сервисов настроен непрерывный мониторинг состояния здоровья.
* **Централизованные логи (XI фактор)**: Логи PHP-FPM перенаправлены в `stdout/stderr` для автоматического сбора оркестратором.

## 🏗 Архитектура проекта
Инфраструктура состоит из семи независимых контейнеров, разделенных на две изолированные сети:

1. **proxy (HAProxy 3.4-alpine)** — входная точка и SSL-терминатор. Принимает трафик на портах `8080` (HTTP, с редиректом) и `8443` (HTTPS). Распределяет запросы между тремя веб-нодами по алгоритму Round Robin. На порту `8404` доступна панель статистики. Подключен к `frontend-net`.
2. **web1, web2, web3 (Nginx 1.26-alpine)** — три параллельных независимых веб-сервера для обеспечения высокой доступности. Отдают статику и проксируют PHP-запросы на бэкенд. Подключены к сетям `frontend-net` и `backend-net`, выступая мостом.
3. **php-fpm (PHP 8.3-fpm-alpine)** — выделенный stateless-интерпретатор PHP. Изолирован в сети `backend-net`. Обрабатывает динамический код, хранит сессии в Redis и выполняет запросы к БД через PDO MySQL.
4. **redis (Redis 7-alpine)** — резидентное хранилище сессий. Обеспечивает единое состояние (state) для всех веб-нод. Порты наружу закрыты, доступен только в `backend-net`.
5. **db (MariaDB 10.11)** — СУБД. Изолирована в сети `backend-net`. Данные сохраняются в постоянном волюме `db_data`.

## 📁 Структура каталогов
* `haproxy/` — конфигурация балансировщика (`haproxy.cfg`).
* `haproxy/certs/` — объединенный самоподписанный SSL-сертификат (`server.pem`).
* `nginx/` — настройки виртуальных хостов Nginx (`default.conf`).
* `php-fpm/` — `Dockerfile` для сборки кастомного образа PHP с расширениями `pdo_mysql`, `redis` и настроенным логированием в stdout.
* `web-site/` — динамический исходный код веб-страниц (`index.php`).
* `.env` — файл конфигурации с секретами (игнорируется Git).
* `.env.example` — шаблон переменных окружения для разработки.
* `podman-compose.yml` — главный декларативный манифест оркестрации.

## 🚀 Быстрый старт

### 1. Подготовка SSL-сертификата на хосте (Fedora)
Сгенерируйте и объедините ключи для HAProxy:
```bash
mkdir -p haproxy/certs
openssl req -x509 -newkey rsa:2048 -nodes -keyout haproxy/certs/key.pem -out haproxy/certs/cert.pem -days 365 -subj "/CN=localhost"
cat haproxy/certs/cert.pem haproxy/certs/key.pem > haproxy/certs/server.pem
rm haproxy/certs/key.pem haproxy/certs/cert.pem
```

### 2. Настройка конфигурации (12-Factor)
Создайте локальный файл конфигурации из шаблона и заполните его своими паролями:
```bash
cp .env.example .env
nano .env
```

### 3. Запуск инфраструктуры со сборкой образов
```bash
podman-compose up -d --build
```

### 4. Проверка статуса и здоровья (Health-checks)
Убедитесь, что все контейнеры перешли в статус `(healthy)`:
```bash
podman-compose ps
```

## 📊 Проверка работоспособности
* **Просмотр сайта (HTTPS)**: Откройте в браузере `https://localhost:8443`. При обновлении страницы (F5) счётчик посещений будет стабильно увеличиваться (благодаря Redis), несмотря на то, что запросы распределяются балансировщиком по разным веб-нодам Nginx.
* **Автоматический L7-редирект**: Введите в браузере `http://localhost:8080`. Балансировщик автоматически выполнит 301-редирект на безопасный порт `https://localhost:8443`.
* **Панель мониторинга HAProxy**: Доступна в браузере по адресу `http://localhost:8404`. Там же можно наблюдать за статусом прохождения health-checks для каждой ноды.
* **Тестирование отказоустойчивости**: Остановите одну из нод `podman-compose stop web1`. Приложение продолжит работать без потери сессии и данных.

## 🔒 Особенности реализации (Rootless & Security)
* Проект полностью работает в rootless режиме без прав суперпользователя.
* Для монтирования конфигурационных файлов использован флаг `:Z` и режим `:ro` (Read-Only) для защиты от изменений изнутри контейнеров.
* Для папки проекта `./web-site/` применен разделяемый флаг **`:z`** (строчная буква), что обеспечивает одновременный безопасный доступ со стороны SELinux для нескольких контейнеров (`web1`, `web2`, `php-fpm`).
